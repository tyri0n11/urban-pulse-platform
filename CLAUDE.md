# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Start the full dev stack (Redpanda, MinIO, app services)
make dev

# Stop all containers
make down

# Lint (ruff)
make lint

# Type-check (mypy, strict mode)
make typecheck

# Run all tests with coverage
make test

# Run only unit or integration tests
make test-unit
make test-integration

# Run a specific test file
uv run pytest apps/ingestion/tests/test_orchestrator.py

# Regenerate routes.json from zones.json
python routes_generator.py

# Build Docker images
make build
make build-serving
```

All Python commands must be prefixed with `uv run`. The project uses a `uv` workspace — do not use `pip` or `python` directly.

## Architecture

This is a **Python monorepo** (uv workspace) for a real-time urban data platform monitoring traffic, weather, and air quality in the Ho Chi Minh City metro region.

### Monorepo layout

```
apps/           # Deployable services (each is a uv workspace member)
  ingestion/    # Polls external APIs, publishes to Redpanda topics
  streaming/    # Consumes Redpanda topics, applies real-time processing, writes to MinIO
  batch/        # Scheduled DuckDB jobs: bronze → silver → gold medallion promotion
  ml/           # Model training (IsolationForest, Z-score detectors) + MLflow tracking
  serving/      # FastAPI REST API for anomalies and metrics queries

packages/       # Internal shared libraries
  core/         # Pydantic domain models (traffic, weather, anomaly, zones), config
  infra-clients/ # Client factories for Kafka/Redpanda, MinIO, DuckDB, MLflow
  observability/ # Prometheus metrics, structured logging, OpenTelemetry tracing

infra/
  docker/       # docker-compose.base.yaml + dev/prod/test overlays
  monitoring/   # Prometheus config, Grafana dashboards (traffic-overview, anomaly-detection, system-health), Loki

platform/
  schemas/      # JSON Schema definitions for Kafka message formats
```

### Data flow

```
External APIs (VietMap traffic, Open-Meteo, WAQI)
  └─► ingestion  ──► Redpanda topics
                       └─► streaming ──► MinIO /bronze
                                           └─► batch (bronze→silver→gold)
                                                 └─► ml (train on gold, log to MLflow)
                                                       └─► serving (FastAPI reads DuckDB gold layer)
```

### Medallion layers in MinIO

- **bronze**: raw JSON events as received from sources
- **silver**: cleaned, validated, typed records
- **gold**: aggregated analytics tables consumed by the serving API and Grafana

### Geographic domain

The platform models 6 zones around Ho Chi Minh City (`zones.json`): Urban Core, Eastern Innovation, Northern Industrial, Southern Port, Western Peri-urban, Southern Coastal. `routes.json` enumerates directed inter-zone routes and is generated by `routes_generator.py` from `zones.json`.

### Dependency rules

- All apps depend on `packages/core` (domain models + config).
- Apps that need infrastructure use `packages/infra-clients`.
- Apps that need observability use `packages/observability`.
- Packages do not depend on each other.

### Toolchain

- **Package manager / workspace**: `uv`
- **Build backend**: `hatchling`
- **Linter**: `ruff` (line-length 100, target py312)
- **Type checker**: `mypy` (strict)
- **Test runner**: `pytest` with `unit` and `integration` markers
- **Python requirement**: ≥ 3.12
